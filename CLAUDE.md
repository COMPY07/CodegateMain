# Vibe Studio 프로젝트 가이드

## 프로젝트 개요

Vibe Studio는 자연어 요청, 라이브 요소 선택, AI 에이전트, 코드·보안 검수를 결합한 개발 도구다.

현재 `mvp`는 React 18 + Vite 기반 프론트엔드 프로토타입이다. UI 일부는 동작하지만 AI API, 실제 파일 수정, dev server 제어, 보안 스캔은 목업 상태다. 작업 목록과 우선순위는 `TASKS.md`를 따른다.

## 구조와 실행

```text
ideas/   아이디어 문서
mvp/     React/Vite 프론트엔드
pdfs/    기획·기술 문서
TASKS.md 작업 목록
```

```bash
cd mvp
npm install
npm run dev
npm run build
```

## 개발 규약

- 기존 함수형 컴포넌트와 React hooks 스타일을 유지한다.
- 표시 컴포넌트와 데이터·실행 로직을 분리한다.
- 목업 기능은 실제 연동처럼 표현하지 않는다.
- 기존 사용자 변경과 관련 없는 파일은 수정하지 않는다.
- 새 기능은 로딩, 빈 상태, 실패 상태를 함께 고려한다.
- UI 버튼에는 접근 가능한 이름을 제공한다.
- API 키와 secret을 프론트나 저장소에 노출하지 않는다.
- iframe 메시지는 origin과 payload를 검증한다.
- 파일 작업은 허용된 프로젝트 루트 안으로 제한한다.

## 협업 규약

- 작업 전 `git status`와 `TASKS.md`를 확인한다.
- 작업 범위가 크면 프론트, 데이터, API 작업으로 나눈다.
- 관련 없는 포맷 변경이나 리팩터링을 섞지 않는다.
- 완료 후 `npm run build`와 관련 테스트를 실행한다.
- 완료한 작업은 `TASKS.md`에 반영한다.
- 변경 파일, 검증 결과, 남은 제약을 인수인계한다.

## 커뮤니케이션 스타일

- 반말하지 않고 존댓말을 사용한다.
- 핵심부터 간결하게 말한다.
- 확인된 사실과 추측을 구분한다.
- 문제를 보고할 때 원인, 영향, 다음 조치를 함께 제시한다.

## 완료 기준

- 요구한 사용자 흐름이 동작한다.
- 기존 주요 기능에 회귀가 없다.
- 빌드와 관련 테스트가 통과한다.
- 실제 연동 범위와 남은 목업이 명확하다.
- 문서와 `TASKS.md`가 현재 상태와 일치한다.
