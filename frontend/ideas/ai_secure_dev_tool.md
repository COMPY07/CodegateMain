# AI Secure Dev Tool

## 배경 / 문제 의식
- 현재 바이브 코딩(vibe coding)을 하는 개발자가 급격히 늘어나고 있음.
- AI가 개발을 대신 해주는 과정에서 보안 문제 등 놓치는 부분이 매우 많음.
- 그 결과 바이브 코딩된 서비스들이 파도처럼 쏟아져 나오지만, 취약점이 많아 실제로 피해가 발생하거나 유저 정보가 유출되는 등의 문제가 심각하게 발생하고 있음.

## 아이디어 개요
- Codex App, Claude(Claude Code)처럼 동작하는 개발용 앱/도구를 하나 만든다.
- 여기에 우리(팀)의 하네스 및 루프 기반 에이전틱 로직을 적용하고, 그 중간에 **보안 검증 단계**를 추가한다.
- 즉, AI가 자율적으로 개발을 진행하는 과정 중간에 "자발적인 개발 보안"을 책임지는 단계를 끼워 넣는 서비스.

## 동작 방식
- 개발 대상이 웹, 앱, 일반 소프트웨어 등 다양할 수 있음.
- 사용자가 개발 중인 프로젝트의 종류를 선택하면, 그에 맞는 보안 기능을 활성화하고 알맞은 보안 프로토콜을 적용.

### 예시 시나리오
- **React 기반 웹 프로젝트**
  - React 관련 취약점, 프론트엔드 취약점을 찾아내는 데 집중.
  - 퍼징(fuzzing)을 중간 단계에 엮어서 함께 수행.
- **Java 백엔드 서버 프로젝트**
  - 과한 기술 적용으로 인해 불필요한 DTO 파일 등이 과도하게 생성되어 데이터 관리가 어려워지는 등의 구조적 문제를 지적.
  - 더 나은 방향으로 설계를 수정하도록 권고하는 등의 조치를 취함.

## 핵심 가치
- 단순 코드 생성이 아니라, 생성 과정 중간에 보안 검증 및 아키텍처 검토를 자동으로 개입시켜, 바이브 코딩으로 인한 보안 사고를 사전에 예방하는 것이 목표.
- 프로젝트 유형(웹/앱/소프트웨어, 프레임워크·언어 등)에 따라 맞춤형 보안 프로토콜을 동적으로 적용.
