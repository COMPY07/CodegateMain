# Web Hacker Tracker

## 아이디어 개요
- 웹 서비스에서 사용자의 행동 데이터(마우스 위치, 스크롤 위치, 방문 페이지, 페이지 체류 시간 등)를 서버 측에서 트래킹.
- 수집된 행동 데이터를 분석하여 해당 사용자의 행동이 "해커의 행동인지"를 판별하는 프로그램/서비스.

## 트래킹 대상 데이터
- 마우스 위치(움직임 패턴)
- 스크롤 위치
- 방문 중인 페이지(어떤 페이지에 머무는지)
- 페이지별 지속(체류) 시간

## 동작 방식
1. 클라이언트(웹)에서 사용자 행동 데이터를 실시간으로 수집하여 서버로 전송.
2. 서버에서 수집된 데이터를 분석하여 정상 사용자와 해커(비정상/공격) 행동 패턴을 구분.
3. 해커로 판단되는 경우 후속 조치를 자동으로 수행 (예: IP 차단 등).

## 핵심 가치
- 단순 로그 기반 탐지가 아니라 사용자의 실제 행동 패턴(UX 시그널)을 기반으로 이상 행동/공격 시도를 탐지.
- 탐지 이후 IP 차단 등 자동화된 후속 대응(response)까지 연결되는 것이 핵심 포인트.
