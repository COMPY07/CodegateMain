# Vibe Studio 작업 목록

현재 `mvp` 프론트엔드를 기준으로 작성한 구현 체크리스트다.

- **FE**: 백엔드 없이 브라우저 안에서 완료할 수 있는 작업
- **BE**: 서버 API 또는 Tauri/Electron 같은 네이티브 브리지가 필요한 작업
- 우선순위는 `P0`(MVP 필수), `P1`(주요 기능), `P2`(개선) 순서다.
- 진행중인 작업은 [-]로, 완료된 작업은 [x]로 표기한다. 즉 작업 시작 시 [-] 표시한 후 작업 종료 시 [x] 표기한다.

## 1. 백엔드가 필요 없는 작업

### P0 — 기존 UI 기능 완성

- [x] **FE-001 영역 선택 도구 구현**
  - 라이브 프리뷰 위에서 드래그해 영역을 선택한다.
  - 선택한 영역의 좌표와 포함된 요소 정보를 프롬프트 칩에 추가한다.
  - 질문 모드와 영역 선택 모드가 동시에 활성화되지 않게 한다.

- [-] **FE-003 반응형 미리보기 구현**
  - 데스크톱, 태블릿, 모바일 프리셋을 제공한다.
  - 선택한 프리셋에 따라 iframe 너비를 변경한다.
  - 현재 viewport 크기를 화면에 표시한다.

- [x] **FE-004 프리뷰 새로고침 구현**
  - 새로고침 버튼으로 현재 iframe 문서를 다시 로드한다.
  - 질문 모드 상태가 새로고침 후에도 다시 적용되게 한다.

- [x] **FE-005 모델 선택 상태 보완**
  - `등록 안됨` 모델은 선택되지 않도록 비활성화한다.
  - 모델 연결 메뉴는 백엔드 연동 전까지 안내 모달을 표시한다.
  - 선택 모델을 새로고침 후에도 유지하도록 `localStorage`에 저장한다.

- [x] **FE-006 세션 로컬 관리 완성**
  - 세션 생성, 이름 변경, 삭제 기능을 구현한다.
  - 세션과 대화를 `localStorage`에 저장한다.
  - 마지막 세션 삭제와 대기 중 응답이 있는 세션 삭제를 안전하게 처리한다.

- [x] **FE-007 프롬프트 칩 다중 선택 보완**
  - 여러 요소를 선택했을 때 모든 칩을 전송 메시지에 표시한다.
  - 동일한 label이 아니라 selector를 기준으로 중복을 판단한다.
  - 칩 순서 변경 및 전체 삭제를 지원한다.

### P1 — 로컬 데이터 뷰어

- [x] **FE-008 CSV 로컬 파일 기능 구현**
  - 사용자가 브라우저에서 CSV 파일을 선택하거나 드래그해 불러온다.
  - 정렬, 검색, 열 너비 조절 기능을 제공한다.
  - 변경 내용을 CSV 파일로 다시 다운로드할 수 있게 한다.
  - 서버에 저장하거나 프로젝트 파일을 직접 수정하는 기능은 범위에서 제외한다.

- [x] **FE-009 PDF 로컬 미리보기 구현**
  - 로컬 PDF 파일 선택 및 드래그앤드롭을 지원한다.
  - 페이지 이동, 확대/축소, 맞춤 보기 기능을 제공한다.
  - 서버의 PDF 목록 조회 및 저장은 범위에서 제외한다.

- [x] **FE-010 대시보드 컴포넌트 개선**
  - 목업 데이터 기준 기간 필터와 툴팁을 구현한다.
  - 빈 데이터, 로딩, 오류 상태 UI를 만든다.
  - 추후 API 데이터를 받을 수 있도록 표시 컴포넌트와 데이터 계층을 분리한다.

- [x] **FE-011 파일 트리 UI 기능 보완**
  - 목업 데이터 기준 파일 선택, 활성 상태, 검색, 전체 펼치기/접기를 구현한다.
  - 파일 내용 수정이나 실제 디스크 반영은 포함하지 않는다.

- [x] **FE-012 에이전트 진행 화면 컴포넌트화**
  - 정적 데이터로 완료, 진행, 대기, 실패, 취소 상태를 모두 표현한다.
  - 실행이 없을 때와 연결이 끊겼을 때의 화면을 추가한다.
  - 실제 실행 상태 수신은 BE 작업으로 분리한다.

### P2 — 품질 및 사용성

- [x] **FE-013 URL hash 동기화 개선**
  - 탭과 질문 모드 변경 시 URL hash도 갱신한다.
  - 브라우저 뒤로가기/앞으로가기로 이전 상태를 복원한다.

- [x] **FE-014 모바일 레이아웃 구현**
  - 작은 화면에서 좌우 패널을 drawer 형태로 전환한다.
  - 탭과 툴바가 넘칠 때 스크롤 또는 축약 UI를 제공한다.

- [-] **FE-015 접근성 개선**
  - 아이콘 버튼에 접근 가능한 이름을 제공한다.
  - 모델 메뉴, 세션, 파일 트리를 키보드로 조작할 수 있게 한다.
  - focus 스타일과 명도 대비를 점검한다.

- [x] **FE-016 프론트 테스트 구성**
  - Vitest와 React Testing Library를 설정한다.
  - 질문 모드, 칩, 세션, 모델 선택의 컴포넌트 테스트를 작성한다.
  - Playwright로 주요 사용자 흐름 E2E 테스트를 추가한다.

- [x] **FE-017 오류 처리 및 알림 UI**
  - 공통 toast, 확인 dialog, error boundary를 구현한다.
  - 이후 BE API 오류를 동일한 UI로 표시할 수 있게 인터페이스를 정의한다.

- [x] **FE-018 사용자 테마 편집 기능**
  - 설정의 테마 화면에서 강조 색상과 채도를 실시간으로 조절한다.
  - 밝은/어두운 모드와 배경·패널·스크롤바의 전체 색조를 함께 변경한다.
  - 선택한 테마를 `localStorage`에 저장하고 앱 전체 UI에 적용한다.
  - 기본값 초기화와 현재 색상 미리보기를 제공한다.

## 2. 백엔드 또는 네이티브 브리지가 필요한 작업

### P0 — 핵심 실행 경로

- [-] **BE-001 프로젝트 파일시스템 연동**
  - 실제 프로젝트 디렉터리와 파일 목록을 조회한다.
  - 파일 읽기, 생성, 수정, 삭제 API를 제공한다.
  - 경로 탈출 방지, 허용 루트 검증, 변경 전 diff 확인을 적용한다.
  - 웹 서버 API 또는 Tauri/Electron 파일시스템 브리지가 필요하다.

- [ ] **BE-002 실제 라이브 프리뷰 연결**
  - 선택한 프로젝트의 dev server를 실행하고 종료한다.
  - 실제 프리뷰 URL과 실행 상태를 프론트에 전달한다.
  - 빌드 오류와 런타임 로그를 수집하고 hot-reload 상태를 표시한다.

- [ ] **BE-003 AI 모델 호출 게이트웨이 구현**
  - Claude, OpenAI 등 모델별 요청 형식을 통합한다.
  - 응답을 SSE 또는 WebSocket으로 스트리밍한다.
  - API 키가 브라우저에 노출되지 않게 서버에서 관리한다.
  - 모델별 오류, timeout, rate limit을 처리한다.

- [ ] **BE-004 프롬프트에서 실제 코드 수정까지 연결**
  - 질문 모드가 전달한 label과 selector를 모델 컨텍스트에 포함한다.
  - 변경 대상 파일을 탐색하고 patch를 생성한다.
  - 사용자 승인 후 patch를 실제 파일에 적용한다.
  - 적용 결과와 실패 원인을 세션 대화로 반환한다.

- [ ] **BE-005 에이전트 실행 오케스트레이션**
  - 작업 분해, 서브에이전트 실행, 의존성, 취소를 관리한다.
  - 단계별 상태와 진행률을 실시간으로 프론트에 전송한다.
  - 현재 목업 `agentRun` 데이터를 실제 실행 이벤트로 교체한다.

### P1 — 보안 및 도구 연동

- [ ] **BE-006 로컬 코드 검수 구현**
  - 변경 diff 또는 대상 함수를 분석한다.
  - 보안, 누락, 중복, 품질 문제를 구조화된 결과로 반환한다.
  - 검수 결과를 코드 수정 루프에 포함한다.

- [ ] **BE-007 취약점 스캔 구현**
  - 프로젝트 종류에 따라 적합한 스캐너를 실행한다.
  - dependency, secret, 정적 분석 결과를 통합한다.
  - 발견 위치, 심각도, 수정 제안을 프론트에 전달한다.
  - 외부 명령 실행 대상과 인자를 allowlist로 제한한다.

- [ ] **BE-008 MCP 도구 연결 및 실행**
  - MCP 서버 등록, 연결 상태 확인, 도구 목록 조회를 제공한다.
  - 도구 호출 권한과 사용자 승인을 관리한다.
  - 실행 결과와 로그를 해당 에이전트 단계에 연결한다.

- [ ] **BE-009 모델/API 키 관리**
  - API 키 등록, 수정, 삭제, 연결 테스트 API를 제공한다.
  - 키를 암호화하거나 OS keychain에 저장한다.
  - 등록 여부와 사용량만 프론트에 전달하고 원문 키는 반환하지 않는다.

- [ ] **BE-010 실제 사용량 대시보드**
  - 작업 수, 토큰 사용량, 비용, 검수 및 보안 결과를 저장한다.
  - 기간별 집계 API를 제공한다.
  - 현재 목업 차트 데이터를 API 응답으로 교체한다.

### P2 — 저장 및 운영

- [ ] **BE-011 세션 및 대화 영구 저장**
  - 세션과 메시지, 선택 요소, 실행 결과를 저장한다.
  - 세션 목록, 상세 조회, 수정, 삭제 API를 제공한다.
  - 스트리밍 중 재접속 시 진행 상태를 복구한다.
  - 기본 이름(`New Agent {번호}`)인 새 세션은 첫 채팅 내용을 요약해 관련 이름으로 자동 변경한다.

- [ ] **BE-012 서버 파일 CSV/PDF 연동**
  - 프로젝트 또는 서버에 있는 CSV/PDF 파일 목록과 내용을 제공한다.
  - 권한이 허용된 파일만 읽고 저장할 수 있게 한다.
  - FE-008, FE-009의 로컬 파일 모드와 서버 파일 모드를 구분한다.

- [ ] **BE-013 실행 로그 및 감사 기록**
  - 모델 호출, 도구 실행, 파일 변경, 승인 기록을 남긴다.
  - 민감 정보는 로그에서 마스킹한다.
  - 세션별 로그 조회와 다운로드 API를 제공한다.

## 3. 권장 구현 순서

1. `FE-001` ~ `FE-007`로 현재 프로토타입의 끊긴 UI 동작을 먼저 완성한다.
2. `BE-001` ~ `BE-004`로 파일 수정이 실제 프리뷰에 반영되는 단일 경로를 만든다.
3. `BE-005` ~ `BE-009`로 에이전트, 보안 검수, 모델 관리 기능을 연결한다.
4. 나머지 데이터 뷰어, 대시보드, 저장, 테스트와 접근성을 보강한다.

## 4. 범위 판단 기준

| 기능 | 백엔드 불필요 범위 | 백엔드 필요 범위 |
|---|---|---|
| 세션 | 브라우저 상태 및 `localStorage` 저장 | 서버 영구 저장, 장기 보관 |
| 파일 트리 | 목업 목록 표시와 선택 | 실제 디스크 탐색, 파일 읽기/수정 |
| CSV/PDF | 사용자가 직접 선택한 로컬 파일 보기 | 서버·프로젝트 파일 조회와 저장 |
| 라이브 프리뷰 | 정적 HTML 또는 사용자가 입력한 URL 표시 | dev server 실행, 로그, hot-reload 관리 |
| 모델 선택 | 모델 목록 UI와 로컬 선택 상태 | 실제 모델 호출, API 키, 사용량 집계 |
| 에이전트 CoT | 목업 상태 렌더링 | 실제 에이전트 실행과 실시간 상태 수신 |
| 보안 도구 | 결과 화면과 빈 상태 UI | 스캐너 실행, 결과 수집, 자동 수정 |
